The Problem

Tech Mahindra's enterprise clients — logistics companies, government fleet operators, and delivery networks — were managing 10,000+ vehicles with no centralized visibility. Route deviations weren't caught until hours later. Fuel theft was undetected. Drivers had no accountability without real-time oversight.

The challenge wasn't just building a GPS app — it was building a platform that could reliably ingest high-frequency telemetry from thousands of hardware devices simultaneously, process geofence violations in real-time, and present actionable intelligence to dispatchers and executives through a single web interface.

📍

High-Frequency Telemetry

GPS devices ping every 10–30 seconds. At 15K vehicles, that's 30–90K pings/minute requiring reliable ingestion.

⚡

Sub-Second Live Maps

Fleet operators demanded live map updates under 2 seconds — polling was out. A persistent push architecture was essential.

🗺️

Geofence Processing

Each ping needed real-time validation against hundreds of geofence polygons per vehicle — computationally expensive at scale.

📊

Historical Analytics

Reporting on billions of historical rows for route replay, mileage, idle time, and SLA analysis without degrading live performance.

🏗️
System Architecture

I designed a dual-pipeline architecture: a high-throughput write path optimized for bulk telemetry ingestion, and a read path optimized for real-time queries and dashboard serving — completely decoupled to prevent analytics from impacting live tracking.

Engineering Challenges Solved

01

Bulk Telemetry Ingestion Without Bottlenecks

Standard row-by-row inserts couldn't handle the throughput. I implemented SQL Server Table-Valued Parameters (TVPs) with batched async inserts, grouping incoming pings into 500ms windows before committing to the DB. This reduced DB write round-trips by 95% and eliminated the ingestion bottleneck.

02

Real-Time Geofence Validation at Scale

Validating each GPS point against hundreds of polygon geofences per vehicle was computationally expensive. I implemented a spatial indexing strategy using SQL Server's geography type with bounding-box pre-filter + precise polygon check only for candidates, reducing compute per event by 85%.

03

Sub-2-Second Live Map Updates

Built a SignalR hub with vehicle-group scoping — dispatchers only receive updates for their assigned fleet segments. Combined with Redis pub/sub for cross-server broadcasting in multi-node IIS deployment, live updates consistently achieved <1.2s end-to-end latency in production.

04

Table Partitioning for Historical Queries

With billions of historical telemetry rows, reporting queries were timing out. Implemented SQL Server partition switching on the telemetry table (monthly partitions), with a separate indexed summary table for dashboard queries. Report generation dropped from 45+ seconds to under 3 seconds.

05

Device Protocol Parsing

GPS hardware used 3 different proprietary protocols alongside standard NMEA. Built a pluggable protocol parser pipeline using the Strategy pattern, where each parser is registered at startup and selected based on device identifier — making new device types a 1-day integration.
